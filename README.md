# Mars One — учебный архив Яндекс Лицея

Учебный проект на Flask, созданный во время обучения в Яндекс Лицее. Репозиторий сохраняет путь от небольших упражнений с маршрутами до приложения с базой данных, формами и авторизацией.

Основная точка входа в портфолио — [`app.py`](app.py). Более ранние файлы сохранены как исторические этапы и не переписаны.

## Что демонстрирует проект

- маршрутизацию Flask и динамические параметры URL;
- переиспользуемые HTML-шаблоны Flask и условный вывод данных;
- создание и валидацию форм с Flask-WTF;
- работу с SQLite через SQLAlchemy;
- регистрацию, хеширование паролей и пользовательские сессии Flask-Login;
- оформление старого учебного проекта в виде адаптивной портфолио-витрины.

## Локальный запуск

Рекомендуется Python 3.10 или новее.

Создайте виртуальное окружение:

```bash
python -m venv .venv
```

Активируйте его в Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

Установите зависимости и запустите приложение:

```bash
python -m pip install -r requirements.txt
python app.py
```

После запуска откройте <http://127.0.0.1:5000>.

Для разработки рекомендуется задать собственный ключ пользовательских сессий:

```powershell
$env:SECRET_KEY = "замените-на-случайное-значение"
```

## История разработки

| Этап | Исходное задание | Результат |
| --- | --- | --- |
| 1 | `training`, `job list` | Маршруты, параметры URL и условия в HTML-шаблонах Flask |
| 2 | `carousel`, `auto-answer` | Статические файлы, формы и шаблоны |
| 3 | `protection`, `distribution` | Flask-WTF и валидация данных |
| 4 | `sqlalchemy captain`, `first job` | ORM-модели и база данных SQLite |
| 5 | `journal job`, `register` | Журнал работ, аккаунты и авторизация |
| Оформление портфолио | `app.py` | Единая точка входа и общая витрина заданий |

История Git содержит исходные коммиты, сделанные по мере прохождения занятий. Основные исторические файлы:

- [`main.py`](main.py) — ранние упражнения с шаблонами и формами;
- [`carousel.py`](carousel.py) — исходное задание с маршрутами и галереей;
- [`server.py`](server.py) — итоговое приложение этапа по работе с базой данных;
- [`app.py`](app.py) — поддерживаемая портфолио-версия, объединяющая полезные части проекта.

## Структура проекта

```text
app.py          единое портфолио-приложение
data/           формы, модели SQLAlchemy и настройка сессий
db/blogs.db     исходная учебная база и примеры данных миссии
templates/      общие HTML-шаблоны Flask
static/         стили и изображения
tests/          базовые проверки маршрутов
```

## Проверка

Для установки инструментов разработки выполните:

```bash
python -m pip install -r requirements-dev.txt
```

Запуск тестов:

```bash
python -m pytest
```

### Настройка PyCharm

В качестве интерпретатора проекта выберите виртуальное окружение:

```text
<папка проекта>\.venv\Scripts\python.exe
```

Путь настраивается в разделе `Settings → Project → Python Interpreter`. После смены интерпретатора устанавливать пакеты в глобальный Python не требуется.

Если появляется ошибка `cannot import name 'url_decode'`, тесты запущены в старом окружении с несовместимыми версиями Flask-Login и Werkzeug. Установите зависимости именно выбранным интерпретатором:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements-dev.txt
.\.venv\Scripts\python.exe -m pytest
```
